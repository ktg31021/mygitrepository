Linux 명령어 조사

## 개요

Linux 운영체제에서는 실행 중인 프로세스와 작업을 관리하기 위해 다양한 명령어를 제공한다.
이번 보고서에서는 대표적인 프로세스 관리 명령어인 `top`, `ps`, `jobs`, `kill` 명령어에 대해 조사하였다.

---

## 명령어 비교

| 명령어 | 기능 |
|---------|---------|
| top | 실시간 프로세스 상태 확인 |
| ps | 프로세스 목록 출력 |
| jobs | 백그라운드 작업 확인 |
| kill | 프로세스 종료 |

---

# 1. top 명령어

## 설명

top 명령어는 현재 시스템에서 실행 중인 프로세스의 상태를 실시간으로 보여주는 명령어이다.

CPU 사용률, 메모리 사용량, PID(Process ID), 사용자 정보 등을 확인할 수 있다.

## 사용법

```bash
top
```

## 주요 정보

- PID : 프로세스 번호
- USER : 실행 사용자
- %CPU : CPU 사용률
- %MEM : 메모리 사용률
- COMMAND : 실행 중인 명령어

## 장점

- 실시간 모니터링 가능
- 시스템 상태를 한눈에 확인 가능

## 실행 예시

```bash
top
```


---

# 2. ps 명령어

## 설명

ps(Process Status)는 현재 실행 중인 프로세스 정보를 출력하는 명령어이다.

top과 달리 명령 실행 시점의 정보를 한 번만 출력한다.

## 사용법

```bash
ps
```

현재 터미널에서 실행 중인 프로세스 확인

```bash
ps -ef
```

전체 프로세스 확인

```bash
ps aux
```

상세 정보 출력

## 주요 옵션

| 옵션 | 설명 |
|--------|--------|
| -e | 전체 프로세스 출력 |
| -f | 상세 정보 출력 |
| aux | BSD 형식 전체 출력 |

## 실행 예시

```bash
ps -ef
```


---

# 3. jobs 명령어

## 설명

jobs 명령어는 현재 쉘에서 실행 중인 백그라운드 작업(Job)을 출력한다.

백그라운드 작업이란 프로그램을 뒤에서 실행시키는 작업을 의미한다.

## 사용법

```bash
jobs
```

## 예시

백그라운드 실행

```bash
sleep 100 &
```

작업 확인

```bash
jobs
```

출력 예시

```text
[1]+ Running sleep 100 &
```

## 특징

- 현재 쉘에서 관리되는 작업만 표시
- 백그라운드 작업 확인 가능



---

# 4. kill 명령어

## 설명

kill 명령어는 특정 프로세스를 종료할 때 사용한다.

프로세스 번호(PID)를 이용하여 종료한다.

## 사용법

```bash
kill PID
```

예시

```bash
kill 1234
```

강제 종료

```bash
kill -9 1234
```

## 주요 시그널

| 시그널 | 의미 |
|----------|----------|
| 15 (SIGTERM) | 정상 종료 |
| 9 (SIGKILL) | 강제 종료 |
| 2 (SIGINT) | 인터럽트 |

## 실행 예시

```bash
ps -ef
kill 1234
```
